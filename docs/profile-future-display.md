# 玩家檔案：未來內容變多時的管理與顯示

當**喜好**、**徽章**、**收藏**等區塊數量變多，可用以下方式管理與顯示，避免一頁過長、難找重點。

**規模假設**：遊戲總數約 **8000 款**；玩家檔案將新增 **「我的收藏」** 區塊，數量可能非常多。整體擴充策略（含編輯頁、搜尋 API）見 [scaling-profile-and-edit.md](./scaling-profile-and-edit.md)。

---

## 一、目前結構（簡述）

- **頂部卡片**：頭貼、暱稱、地區、社群、MBTI、簡介、桌遊偏好(8軸)、按鈕、統計
- **我的桌遊喜好**：分頁（超喜歡 / 喜歡 / 普通 / 不喜歡 / 想玩 / 沒興趣），每頁遊戲卡片
- **探索清單**、**品味最接近 / 最獨特玩家**
- **我的成就徽章**：一牆格子
- **我的收藏**（規劃中）：獨立卡片區塊，數量可能超級多，需與喜好同一套顯示策略

---

## 二、建議策略總覽

| 區塊 | 變多時怎麼處理 | 建議方式 |
|------|----------------|----------|
| **喜好（各分頁遊戲數）** | 每分頁幾十～上百款 | **分頁（每頁 N 張）** 或 **「顯示更多」摺疊** |
| **我的收藏** | 可能超級多 | **前 N 張 + 顯示更多／分頁**（與喜好同一套，一上線就做） |
| **徽章** | 成就種類變多 | **依稀有度篩選** + **預設摺疊／展開** |
| **整頁區塊** | 區塊很多（喜好、收藏、徽章、清單、活動…） | **大區塊分頁／錨點** 或 **手風琴摺疊** |

---

## 三、各區塊具體做法

### 1. 喜好（我的桌遊喜好）

- **現狀**：已有分頁（超喜歡 / 喜歡 / …），但每個分頁內是「整批渲染」。
- **變多時**：
  - **方案 A**：每個分頁內再做 **每頁 12～24 張** 的小分頁（上一頁 / 下一頁），或「載入更多」按鈕。
  - **方案 B**：預設只顯示前 N 張（例如 12），其餘放在「展開顯示全部」裡（點擊再展開或再載入）。
- **建議**：先做 **方案 B（顯示前 N + 展開更多）**，實作簡單；若單一分頁常破 50 款再加方案 A 分頁。

### 2. 我的收藏（新區塊）

- **設計前提**：此區塊從一開始就當作「**可能超級多**」，與喜好同一套策略，避免上線後才因數量爆量改版。
- **做法**：
  - **前 N 張（例如 12～24）+ 「顯示更多」按鈕**，點擊可展開下一批或分頁載入。
  - 若單區常破百，可做**分頁**（每頁 N 張）或**虛擬捲動**（見 [scaling-profile-and-edit.md](./scaling-profile-and-edit.md)）。
- **API**：若後端「我的收藏」為獨立表或陣列，建議支援「只回傳前 M 筆 + total」，前端「載入更多」時再要下一批，避免一次回傳數千筆。

### 3. 徽章

- **現狀**：一牆 `repeat(auto-fill, minmax(130px, 1fr))`，沒有篩選。
- **變多時**：
  - **篩選**：依稀有度（傳說 / 史詩 / 稀有 / 一般）下拉或 tab，只顯示該類。
  - **摺疊**：  
    - 預設只顯示「已解鎖」或「最近 N 個」；其餘收在「顯示全部」摺疊。  
    - 或依「類型」（例如系統成就 / 活動成就）分組，每組可摺疊。
- **建議**：先加 **稀有度篩選**（下拉或 tab），再視數量決定是否做「預設只顯示部分 + 展開全部」。

### 4. 整頁區塊（喜好、收藏、徽章、清單、未來活動…）

- **現狀**：一頁直直排下去。
- **變多時**：
  - **方案 A：錨點導覽**  
    頂部或側邊固定「喜好｜徽章｜清單｜…」連結，點擊捲動到對應區塊。不減少 DOM，只改善導覽。
  - **方案 B：大區塊分頁**  
    例如「概覽 / 喜好 / 徽章 / 清單」各成一個分頁，一次只顯示一個大區塊。
  - **方案 C：手風琴（摺疊）**  
    每個大區塊有標題列，點擊可摺疊／展開；預設可只展開「概覽 + 喜好」，其餘收合。
- **建議**：  
  - 區塊 < 5 個：**錨點導覽**即可。  
  - 區塊 ≥ 5 或單區塊很長：可考慮 **手風琴** 或 **大區塊分頁**，依你們偏好的閱讀習慣選一種。

---

## 四、實作優先順序建議

1. **短期**  
   - 喜好：每分頁內「只顯示前 12～24 張 + 顯示更多／展開」避免一次渲染過多。  
   - 我的收藏：上線時即做「前 N + 顯示更多」或分頁，與喜好同一套。  
   - 徽章：加「稀有度」篩選（下拉或 tab）。

2. **中期**  
   - 頂部或側邊加 **錨點**（#喜好、#徽章、#清單…），方便在長頁面跳轉。  
   - 徽章若再變多：分組（例如依類型）或「預設顯示部分 + 展開全部」。

3. **長期**  
   - 若區塊繼續增加（例如多種清單、活動紀錄）：再引入 **大區塊分頁** 或 **手風琴**，並維持同一套設計語言（例如都用摺疊或都用分頁）。

---

## 五、小結

- **分頁**：適合「同一類、數量多」的列表（例如喜好底下的遊戲、未來評分紀錄）。
- **篩選**：適合「同一區塊、多維度」的內容（例如徽章依稀有度）。
- **摺疊／展開**：適合「區塊多或單區塊很長」的整頁排版，減少一次看到的量。
- **錨點**：成本低，先做就能明顯改善長頁面導覽。

若要開始實作，可從「喜好每分頁前 N 張 + 顯示更多」和「徽章稀有度篩選」兩項先做，再視使用情況加錨點或摺疊。
