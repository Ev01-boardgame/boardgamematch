# 桌遊偏好模組（約 24 題）— 題目與計分設計

> 與主測驗（人格 24 題）一起使用，產出「偏好向度」分數，供性格 × 喜好研究與推薦使用。  
> 題多一點、準確度高一點，總題數約 24 題。

---

## 一、偏好向度定義（計分用）

測驗結果會產出以下向度的分數，可與人格標籤池做相關／迴歸：

| 向度類別 | 向度名稱 | 說明 |
|----------|----------|------|
| **類型** | 派對傾向 | 喜歡熱鬧、多人、簡單、炒氣氛 |
| | 策略傾向 | 喜歡動腦、規劃、最優解、競爭 |
| | 合作傾向 | 喜歡一起破關、團隊目標、低對立 |
| | 劇情／沉浸傾向 | 喜歡故事、角色、情境、代入感 |
| | 家庭／輕量傾向 | 喜歡規則簡單、一局短、易上手 |
| | 抽象／純機制傾向 | 喜歡少主題、重邏輯、耐玩 |
| **機制** | 工人放置 | 放工人、搶行動、資源轉換 |
| | 手牌管理 | 手牌、 combo、牌組構築 |
| | 談判／結盟 | 交易、談判、陣營、嘴砲 |
| | 擲骰／隨機 | 骰子、抽牌、不確定性 |
| | 隱藏身份 | 陣營、推理、說謊、猜身分 |
| | 競標／拍賣 | 出價、資源競標 |
| **互動** | 零互動傾向 | 各玩各的、比誰分高 |
| | 直接對抗傾向 | 打爆對手、針對、戰爭 |
| | 談判結盟傾向 | 交易、聯盟、嘴砲、說服 |
| | 合作傾向 | 共同目標、分工、討論 |
| **隨機性** | 策略控（低隨機） | 喜歡可計算、少運氣 |
| | 運氣接受度（高隨機） | 接受骰運、逆轉、驚喜 |

實作時可簡化：例如類型 6 個、機制 6 個、互動 4 個、隨機 1–2 個維度，或全部攤平為 15–20 個標籤。

## 二、題目與選項（24 題）

下列每題格式：**題目** → 選項與對應加分的向度（括號內為該選項對哪些向度 +1 或 +2，可依需要調整權重）。

---

### 類型偏好（7 題）

**Q1. 朋友聚會時，你比較常想開哪一類遊戲？**
- 規則簡單、大家都能很快上手的 → 派對+1, 家庭輕量+1
- 有策略、要動腦、想贏要算的 → 策略+2
- 大家一起合作破關、打 BOSS 的 → 合作+2
- 有故事、有角色、能代入的 → 劇情沉浸+2
- 規則少、節奏快、笑聲多的 → 派對+2
- 機制特別、想試新東西的 → 抽象/純機制+1（或另設「探索」）

**Q2. 買桌遊時，你最容易被哪種描述吸引？**
- 「派對神作、一局 15 分鐘」→ 派對+2
- 「策略深度高、重開性強」→ 策略+2
- 「純合作、一起解謎」→ 合作+2
- 「有完整劇情與角色」→ 劇情沉浸+2
- 「規則簡單、適合全家」→ 家庭輕量+2
- 「機制創新、沒玩過」→ 抽象/純機制+1

**Q3. 你比較喜歡的遊戲時長是？**
- 一局 15–30 分鐘，快節奏 → 派對+1, 家庭輕量+1
- 一局 45 分鐘–1 小時，剛剛好 → （不特別加，或平衡）
- 一局 1.5–2 小時以上，可以沉進去 → 策略+1, 劇情沉浸+1

**Q4. 以下哪種「類型標籤」最常出現在你的願望清單？**
- 派對、歡樂、多人同樂 → 派對+2
- 策略、歐式、德式 → 策略+2
- 合作、團隊、共鬥 → 合作+2
- 劇情、角色扮演、故事 → 劇情沉浸+2
- 家庭、親子、易上手 → 家庭輕量+2
- 抽象、邏輯、無主題 → 抽象+2

**Q5. 遊戲結束後，你比較會因為什麼而滿足？**
- 大家笑成一團、氣氛很好 → 派對+2
- 我算對了、贏了或雖敗猶榮 → 策略+2
- 我們一起過關了 → 合作+2
- 故事或角色很投入 → 劇情沉浸+2
- 又試了一款新遊戲 → 抽象/探索+1

**Q6. 你更喜歡「一局定勝負」還是「多局累積／戰役」？**
- 一局定勝負、節奏快 → 派對+1, 家庭輕量+1
- 多局累積分數、有延續感 → 策略+1
- 戰役／劇情延續、有成長感 → 劇情沉浸+2
- 看遊戲設計，都可以 → （不計或平衡）

**Q7. 對於「輕度家庭遊戲」你的態度是？**
- 很愛，常開、輕鬆 → 家庭輕量+2
- 可以玩，但更想開策略 → 策略+1
- 不排斥，但更愛派對或合作 → 派對+1 或 合作+1
- 比較少主動選 → （不計）

### 機制偏好（6 題）

**Q8. 你喜歡「工人放置」類的遊戲嗎？（放米寶、搶行動格那種）**
- 非常喜歡，會主動找 → 工人放置+2
- 喜歡，有就玩 → 工人放置+1
- 普通 → （不計）
- 不太喜歡，偏好其他機制 → （不計或 -1，依你要不要做負向）

**Q9. 你喜歡「手牌管理／牌組構築」嗎？（手牌、 combo、升級牌組）**
- 非常喜歡 → 手牌管理+2
- 喜歡 → 手牌管理+1
- 普通／不喜歡 → （不計）

**Q10. 遊戲中有「談判、交易、結盟」時，你的感覺是？**
- 很享受，會主動談 → 談判結盟+2
- 可以接受，有加分 → 談判結盟+1
- 普通 → （不計）
- 不喜歡，想專心算自己的 → （不計）

**Q11. 你對「擲骰、抽牌」這種運氣成分的接受度？**
- 很接受，有驚喜感 → 擲骰隨機+2, 運氣接受度+1
- 可以接受，但不要太多 → 擲骰隨機+1
- 偏好少一點運氣 → 策略控+1
- 不喜歡，喜歡可計算 → 策略控+2

**Q12. 「隱藏身份、陣營、推理、說謊」類的遊戲，你喜歡嗎？**
- 非常喜歡 → 隱藏身份+2
- 喜歡 → 隱藏身份+1
- 普通／不喜歡 → （不計）

**Q13. 你喜歡「競標／拍賣」機制嗎？（出價、搶標）**
- 非常喜歡 → 競標+2
- 喜歡 → 競標+1
- 普通／不喜歡 → （不計）

**Q14. 對於「規則書很厚、要學一陣子」的遊戲，你的態度是？**
- 願意花時間學，值得 → 策略+1 或 桌遊純血+1（若有人格標籤）
- 可以接受，但偏好中等複雜 → （不計）
- 比較偏好規則簡單、好上手 → 家庭輕量+1

### 互動強度（6 題）

**Q15. 遊戲中有人「直接針對你」或「把你打爆」時，你通常？**
- 覺得很刺激、這樣才好玩 → 直接對抗+2
- 可以接受，本來就是競爭 → 直接對抗+1
- 不太喜歡，希望少一點針對 → 合作+1 或 零互動+1
- 很不喜歡，偏好大家各算各的 → 零互動+2

**Q16. 你喜歡遊戲中有「談判、交易、嘴砲」嗎？**
- 非常喜歡，會主動聊 → 談判結盟+2
- 喜歡，有加分 → 談判結盟+1
- 普通 → （不計）
- 不喜歡，想專心策略 → （不計）

**Q17. 多人遊戲時，你比較享受？**
- 各自算自己的，最後比誰分高 → 零互動+2
- 可以打來打去、針對、結盟 → 直接對抗+1, 談判結盟+1
- 大家一起討論、合作過關 → 合作+2
- 有陣營、推理、說謊、猜身分 → 隱藏身份+1（可歸在機制或互動）

**Q18. 「合作遊戲」和「競爭遊戲」你更常選？**
- 更常選合作 → 合作+2
- 更常選競爭 → 直接對抗+1 或 策略+1
- 差不多，看場合 → （不計）

**Q19. 遊戲中有人想跟你「交易、結盟」時，你通常？**
- 很享受，會討價還價 → 談判結盟+2
- 可以，有策略考量 → 談判結盟+1
- 不太喜歡，想自己算 → （不計）

**Q20. 你比較喜歡「大家各玩各的、最後比總分」還是「有直接衝突、可以打人」？**
- 各玩各的、比總分 → 零互動+2
- 有衝突、可以打人 → 直接對抗+2
- 看遊戲，兩種都可以 → （不計）

### 隨機性（4 題）

**Q21. 關鍵時刻「骰子搞砸」或「抽牌爆掉」，你的反應通常是？**
- 覺得很好笑、這就是遊戲 → 運氣接受度+2
- 可以接受，下次再來 → 運氣接受度+1
- 會有點煩，偏好少一點運氣 → 策略控+1
- 很不喜歡，偏好可計算 → 策略控+2

**Q22. 你比較喜歡「靠運氣逆轉」還是「靠計算碾壓」？**
- 靠運氣逆轉、有驚喜 → 運氣接受度+2
- 靠計算、步步為營 → 策略控+2
- 都可以 → （不計）

**Q23. 對於「高隨機、每局差異大」的遊戲，你的態度是？**
- 很愛，每局都不一樣 → 運氣接受度+2, 混沌傾向+1（若有）
- 可以接受 → 運氣接受度+1
- 比較偏好可預期、策略感重 → 策略控+2

**Q24. 遊戲結束後，你比較會回想的是？**
- 哪步走錯了、下次可以怎麼優化 → 策略控+2
- 那顆骰子／那張牌好關鍵、好好笑 → 運氣接受度+2
- 大家玩得開不開心、氣氛怎樣 → 派對+1 或 合作+1
- 故事／角色有沒有代入感 → 劇情沉浸+1

## 三、計分與輸出建議

- 每題單選，依選項對應的向度加分（例如 +1 或 +2，上表已標在題後）。
- 全部答完後，對「類型 / 機制 / 互動 / 隨機」各向度加總，得到一組**偏好分數**。
- 可再正規化（例如 0–100）或只取前幾名向度，供：
  - 與人格 10 型、標籤池做交叉分析；
  - 推薦系統（若某向度分高 → 推薦對應類型／機制）。

若要再提高準確度，可：
- 增加每向度的題數（例如類型再多 2–3 題），或
- 部分題目改為「同意程度 1–5」，用加權計分。

## 四、與主測驗的整合方式

- **流程**：主測驗 24 題（人格）→ 接著 24 題偏好模組 → 一次產出「人格類型 + 人格標籤池 + 偏好向度」。
- **資料**：同一使用者一組 ID，存兩份結果（人格 + 偏好），分析時用 ID 串起來做 10 型 × 偏好、標籤池 × 偏好 等。
